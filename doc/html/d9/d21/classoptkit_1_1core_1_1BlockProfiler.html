Detailed Description

The BlockProfiler class utilizes the RAII technique to initiate and conclude profiling for a specific raw event. Profiling commences upon instantiation and persists until the current scope is exited.

Note that block profiler does NOT group raw_events!

Each raw_event creates seperate file_description (fd) to read data and each raw_event is treated seperately.
In cases where CPU performs multiplexing and since each event treated seperately, you cannot gurantee that
events E1 and E2 will record the same instructions.
for more information about grouping

See also
https://man7.org/linux/man-pages/man2/perf_event_open.2.html
